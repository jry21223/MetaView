# Geography Prompt Guidance

## Common

- 必须先建立固定地图框架，再展示时间变化或空间流动。
- 地图方向、区域名称、图例语义和边界参照必须前后一致。
- 区域位置关系不能因动画变形而改变，除非 topic 本身是构造或板块移动。
- 时间锚点必须明确，如季节、年份、地质时期，不能把不同时间层混成同一状态。
- 迁移、洋流、季风、板块运动、贸易流、降水或气温变化都必须有准确方向。
- 区域对比必须基于同一投影与同一参照，不能换底图后直接比较。
- 多层信息如地形、气候、人口、流向需分层表达，不能同时争夺主视图。
- 视觉上的连续位移不能暗示地图框架本身在移动。

## Planner

- 先拆成地图建立、区域标定、时间或机制引入、空间变化展示、结果对比。
- 优先选能清楚体现区域差异和方向变化的底图与范围，不选过大或过细尺度。
- 对每个 scene 标明当前时间锚点、当前区域和当前流向或变化变量。
- 提前识别最易讲错的点，如南北颠倒、区域误标、季节切换、局地与全球尺度混用。
- 若 topic 涉及成因，先规划静态成因图层，再展示动态结果图层。
- 若存在多个区域对比，先固定共同底图，再逐区叠加差异。
- 若内容跨长时间尺度，按关键时期离散拆分，不用模糊连续播放代替阶段变化。
- 若涉及多个流向系统，只保留主导路径，避免箭头网络压满画面。

## Coder

- 底图固定在稳定位置，后续变化通过叠加层、箭头层、标注层逐步展开。
- 区域标签与地理位置绑定，缩放或切换时保持可追踪。
- 时间变化先切换时间标记，再更新降水、温度、风向、流向或边界状态。
- 箭头方向、长度和颜色语义保持一致，不因版面变化改变含义。
- 区域对比采用并列或同图分层，不要频繁切换不同地图基底。
- 气候带、等值线、板块边界、迁移路径等专题层应逐层显现，避免一次全开。
- 若展示构造运动或环流，保留足够静态参照，避免观众误判位移对象。
- 图例变化必须与专题层同步，不能出现图例先改或滞后。

## Critic

- 检查地图方向、区域标签和边界参照是否前后一致。
- 检查是否在固定地图框架建立前就开始展示时空变化。
- 检查时间锚点是否清楚，是否把不同时期混成同一画面结论。
- 检查箭头方向是否正确，是否把流入流出、升降、迁移方向画反。
- 检查区域对比是否使用了不一致的底图或尺度。
- 检查多层信息是否互相遮挡，导致看不清主变化。
- 检查标签漂移、区域误标或图例失配。
- 检查动画是否把底图移动误当成地理对象移动。

## Repair

- 若方向或标注错误，只修正相关标签、指南针或箭头方向。
- 若时间混乱，补时间锚点并把对应变化对齐到正确阶段。
- 若底图不稳，固定地图框架，只重做叠加层动画。
- 若图例失配，统一图例与专题层语义，不改主分镜。
- 若区域对比失真，改成同底图并列或分层对照。
- 若箭头网络过密，删去次要流向，只保留主路径。
- 若关键区域被遮挡，调整局部标签位置或透明度。
- 若成因与结果混讲，补一个静态成因过渡层。
