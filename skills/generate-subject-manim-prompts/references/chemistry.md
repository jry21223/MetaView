# Chemistry Prompt Guidance

## Common

- 物种身份必须稳定，原子种类、键连关系、电荷、相态不能随动画漂移。
- 成键、断键、重排、中间体生成必须遵守化学键逻辑和可行价态。
- 反应前后必须保持原子守恒与电荷守恒，电子转移需有明确去向。
- 机理步骤必须按真实先后展开，不能把多步过程压成直接产物。
- 中间体、过渡表达和局部电荷只能在有依据时出现，不能凭视觉需要发明。
- 若 topic 涉及酸碱、氧化还原、取代、加成、消除等过程，必须保持反应类型逻辑一致。
- 离子、共价分子、配位键、共振式等不同表示层次不能混作同一关系。
- 视觉重排不能暗示键未断先成或电荷无源变化。

## Planner

- 先拆成反应物建立、关键活性位点识别、步骤级机理变化、产物与守恒核对。
- 优先选能清楚暴露成键断键位置和电荷变化的表示法，不选过度简写。
- 对每个 scene 标明当前物种、当前键变化和当前电荷平衡。
- 提前识别最易讲错的点，如电子流方向、离去基离开时机、质子转移顺序、重排前后连接关系。
- 若 topic 是反应方程式而非机理，也要规划守恒核对步骤，避免只给结果。
- 若有多步机理，只保留主链必要步骤，不省略决定产物的关键中间体。
- 若存在条件依赖，如催化剂、溶剂、温度，先单独建立其作用角色。
- 若涉及结构异构或立体化学，提前规划视角与标记，避免连接关系误读。

## Coder

- 分子骨架保持尽量稳定，局部变化只发生在反应中心附近。
- 断键和成键用明确的局部动画区分，避免整体形变掩盖键变化。
- 电荷、孤对电子、电子流箭头必须与相关原子绑定同步更新。
- 多步机理按一步一因果推进，不把连续电子转移压成一次全局替换。
- 反应物、试剂、条件、产物分区稳定，避免身份在画面中漂移。
- 守恒核对可用局部计数或对照标记，放在步骤切换后立即出现。
- 若涉及中间体，保留其足够停留时间，让观众看清结构与电荷。
- 结构重排时优先保持原子标签一致，避免观众误以为更换了原子。

## Critic

- 检查是否出现不可能价态、错误键数或无依据电荷。
- 检查原子、电荷、相态是否前后不守恒。
- 检查电子流方向是否与成键断键结果一致。
- 检查是否跳过决定产物的关键中间体或关键步骤。
- 检查分子重排是否导致原子身份混淆。
- 检查离子、电子、条件标记是否被遮挡而无法核对。
- 检查动画是否把多步机理误画成一步直接转化。
- 检查反应条件或试剂是否被错误赋予了设定外的作用。

## Repair

- 若价态或键数错误，只修相关原子和键，不重画整个分子。
- 若电荷不守恒，补电荷标注或电子流箭头修正来源与去向。
- 若守恒核对缺失，补一个局部对照步骤。
- 若关键中间体被省略，插入最小必要中间帧。
- 若原子身份混淆，补原子标签或颜色绑定。
- 若机理过度压缩，拆开最近的那一步，不重写全流程。
- 若条件作用不清，补在对应步骤旁的局部标注。
- 若遮挡了反应中心，调整局部布局和分层。
